# agent-gateway-basic 控制台方案

本文档定义 L1 `agent-gateway-basic` 的运营界面与调试界面方案。L1 不再只承担网关职责，还承担七层体系的统一运营入口与调试入口。

## 1. 目标
- 为 L2-L7 提供统一运营控制台。
- 为研发、实施、测试、运维提供统一调试工作台。
- 让 L1 成为七层工程的可观测、可配置、可调试入口。

## 2. 定位
- 项目名：`agent-gateway-basic`
- 层级：L1
- 技术：Java + Web Console
- 角色：
- 统一服务入口
- 七层运营控制台
- 七层调试工作台

## 3. 控制台拆分
### 3.1 运营界面
面向：
- 运营人员
- 架构师
- 实施人员
- 平台管理员

目标：
- 看全局状态
- 管配置与策略
- 看配额、审计、质量、可靠性
- 观察 L2-L7 的运行状态

### 3.2 调试界面
面向：
- 研发
- 测试
- 运维

目标：
- 构造请求
- 调试契约映射
- 看调用链与错误定位
- 回放历史请求

## 4. 运营界面功能点
### 4.1 全局总览
- 七层健康状态
- 请求量、成功率、错误率、平均耗时
- 模型调用量与成本摘要
- 知识调用量与质量摘要
- 当前告警与异常事件

### 4.2 服务治理
- 服务注册与上下线
- 路由配置
- 鉴权策略
- 客户端管理
- 配额与限流
- 熔断与降级状态
- 异步任务与回调状态

### 4.3 审计与安全
- 操作审计
- 请求审计
- 权限变更记录
- 风险操作审批记录
- 密钥与访问控制状态

### 4.4 质量与可靠性
- 质量评分
- SLO 与 SLI
- 失败原因分布
- 测试结果汇总
- 发布风险状态
- 技术债跟踪

## 5. 调试界面功能点
### 5.1 请求调试
- 构造网关请求
- 自定义 header、query、body
- 选择目标服务
- 查看契约转换前后内容
- 一键发送

### 5.2 链路调试
- 查看 L1 -> L2/L3/L4/L5/L6/L7 链路
- 查看每一跳请求与响应
- 查看耗时拆解
- 查看错误定位
- 查看重试与熔断状态

### 5.3 回放与复现
- 根据 `request_id` 回放
- 回放最近失败请求
- 比较两次执行结果

### 5.4 契约调试
- 查看原始请求
- 查看 L1 转换后的标准契约
- 查看下游标准结果
- 查看按契约类型聚合的错误码

## 6. 按层覆盖的运营能力
### 6.1 L2 `agent-business-solution`
- 场景列表
- 场景版本
- 场景状态
- 场景调用量
- 场景成功率与失败率
- 场景执行日志
- 场景回放

### 6.2 L3 `atomic-ai-service`
- 能力目录
- 能力版本
- 能力调用量
- 能力耗时
- 能力错误码
- 能力契约调试

### 6.3 L4 `agent-model-runtime`
- 任务队列
- 并发数
- 超时与重试
- 熔断状态
- 路由命中情况
- 成本与吞吐
- 模型执行明细

### 6.4 L5 `agent-knowledge-ops`
- 知识库列表
- 数据导入流水线
- 质量状态
- 审计流水线
- 知识资产授权状态

### 6.5 L6 `agent-model-hub`
- 模型注册表
- 模型版本
- 路由策略输入
- 模型评测结果
- 成本基线
- 效果对比

### 6.6 L7 `agent-platform-foundation`
- 环境状态
- 容器与服务状态
- 存储、消息、缓存状态
- 日志、追踪、告警
- 备份与容灾状态
- 密钥与访问控制状态

## 7. 信息架构
1. `Dashboard`
- 全局总览
- 七层状态
- 告警与风险

2. `Operations`
- 服务治理
- 路由、鉴权、配额
- 审计与审批

3. `Debug Studio`
- 请求调试
- 契约转换
- 链路追踪
- 回放

4. `Scenarios`
- L2 场景运营

5. `Capabilities`
- L3 能力运营

6. `Model Runtime`
- L4 运行时治理

7. `Knowledge Ops`
- L5 知识运营

8. `Model Hub`
- L6 模型治理

9. `Platform`
- L7 基础设施状态

10. `Quality & Reliability`
- 质量评分
- SLO
- 技术债
- 发布风险

## 8. 后端聚合 API
### 8.1 运营接口
- `GET /ops/overview`
- `GET /ops/layers`
- `GET /ops/l2/scenarios`
- `GET /ops/l3/capabilities`
- `GET /ops/l4/runtime`
- `GET /ops/l5/knowledge`
- `GET /ops/l6/models`
- `GET /ops/l7/platform`

### 8.2 调试接口
- `POST /debug/request`
- `GET /debug/trace/{request_id}`
- `POST /debug/replay/{request_id}`

## 9. MVP 界面范围
### 9.1 第一阶段页面
- `Dashboard`
- `Debug Studio`
- `Scenarios`
- `Model Runtime`

### 9.2 第一阶段后端能力
- 复用现有 `/ops/overview`
- 新增分层聚合接口
- 新增调试请求与链路查询接口
- 展示契约转换统计与错误码统计

## 10. 第一阶段验收标准
- L1 控制台可展示七层概览
- 可在界面上调试 `qa/compliance/pricing`
- 可查看契约转换前后结果
- 可查看 L2-L4 关键运行状态
- 可查看审计、指标、错误码与基础质量状态
