# PLANS

本文件定义执行计划的标准结构与约束，确保计划与实际进度可对齐、可审计。

## 计划结构（统一模板）
每个计划文件放在 `docs/exec-plans/active/`，命名建议：
- `YYYYMMDD-<short-title>.md`

模板：
- 背景与目标
- 范围（in / out）
- 成功标准（验收口径）
- 依赖与风险
- 里程碑与时间点
- 任务拆解（Epic → Story → Task）
- 交付物清单（文档、代码、配置、测试）
- 运行/验证步骤
- 变更记录（日期/作者/摘要）

## 计划生命周期
- active: 进行中
- completed: 已完成（从 active 移动）
- tech-debt: 技术债登记

## 进度更新要求
- 任何关键变更必须记录到计划文件
- 每次交付更新“交付物清单”与“变更记录”
- 遇到阻塞必须记录原因与下一步
