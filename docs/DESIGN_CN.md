# 设计

## 目标
设计一个多项目的 agent harness，用于跨七层 AI 体系进行规划、执行、测试与治理。

## 设计目标
- 跨 Java、Python、IaC 的全栈协同编排
- 以增量交付为核心，显式进度跟踪
- 不侵入业务逻辑，仅约束接口与执行契约
- 高风险操作必须人类审批
- 可观测与审计信息对智能体可读

## 非目标
- 替代各层服务或其内部实现
- 构建单体大一统运行时
- 直接接管模型训练或数据集流水线

## 控制平面
1. 知识平面
- 文档即记录系统
- 通过 AGENTS.md 与 docs/ 渐进式披露
- 架构地图与产品规格

2. 控制平面
- 任务规划与优先级
- 审批闸口与审计记录
- 策略执行（配额、权限、风险）

3. 执行平面
- 构建/测试/运行编排
- 启动序列（单层或全栈）
- 环境管理（dev/staging/prod）

4. 观测平面
- 标准化日志/指标/追踪
- 质量评分与门禁
- 失败分析与自动修复钩子

5. 集成平面
- 七层项目元数据注册
- 统一接口契约与适配器
- 依赖图与调用路由元数据

## 核心产物
- AGENTS.md：面向人类与智能体的导航地图
- ARCHITECTURE.md：鸟瞰图与数据/控制流
- features.json：功能矩阵与状态
- incremental.md：里程碑与进度
- docs/：设计、计划、规格、参考、质量

## 最小服务集合
- Registry Service：项目目录、负责人、入口
- Planner Service：任务、里程碑、依赖
- Runner Service：构建/测试/运行编排
- Auditor Service：审批流与审计
- Observer Service：测试结果、日志、质量门禁

## 概念数据模型
- Project：名称、层级、语言、仓库、入口
- Task：id、标题、状态、依赖、产物
- Run：id、项目、命令、状态、日志、时间
- Approval：id、请求、审批人、决策、原因
- Quality Gate：id、标准、状态、证据

## 执行流程（高层）
1. Agent 读取 AGENTS.md 并定位相关文档
2. Planner 生成任务与依赖
3. Runner 执行目标层的构建/测试/运行
4. Observer 汇总信号并更新质量
5. Auditor 记录审批与结果
